Strem widgets, only local!

 

Introduction

This is a mini-implementation of a go-based web service that is designed to help local fragforce orgs get some neat stream widgets without having to rely on an external service.

The system relies on the go script to pull donations from the team api and the user api records, and keeps them locally, while the pages use JS to read those files and display the data accordingly. As this is a general webserver, additional non-data related pages can be created and used as widgets as well.

 

Contents

blinkingtext.html is a simple blinking text widget, configurable in streamlabs via custom css. Check the html comments for use instructions.

scrollingtext.html is scrollable text that includes custom text and the last few donations, made to scroll across a portion of the screen. Text and usage configurable via YAML settings.

Also, all of the configuration for these files can be found in FSSConfig.yml file, in YAML format. See the comments for more information!

videoplay.html (DEPRICATED) is a simple player that auto-plays videos in the video folder, useful for commercial breaks. Configurable via YAML settings. Though this is still usable, its better from a configuration standpoint to use the VLC Source Player in OBS/SLOBS for streams.

donationalert.html (DEPRICATED) is a fragforce branded alert for donations, with a spinning fragforce logo and sound alerts for donations, configurable via YAML Settings. Note that FRAGBOT alerts already has these alerts, so use this as a backup only.
